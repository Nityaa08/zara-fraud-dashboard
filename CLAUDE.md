# Zara eShop — Transaction Pattern Explorer

## Project Context
A frontend-only fraud analytics dashboard for risk analysts at Zara eShop. Lets analysts filter, visualize, and drill into ~900 transactions to identify fraud patterns.

## Stack
- Next.js 14 (App Router), TypeScript, Tailwind CSS, Recharts, date-fns
- No backend — data loaded from `public/transactions.json`
- Deployed to Vercel

## Key Architecture Decisions
- All state lives in `app/page.tsx` using React hooks. Filters flow down to charts/table.
- Chart clicks toggle filters (e.g., clicking a country pie slice filters by that country).
- Transaction detail is a slide-out panel triggered by table row click.
- Data is generated by `scripts/generate-data.js` and committed as static JSON.

## Embedded Fraud Pattern
50 chargebacks from Brazil, credit_card, electronics, concentrated in a 2-hour window on Feb 24 02:00–04:00 UTC, with IP country mismatches (Nigeria/Russia/China). High amounts ($400–$900), clustered card range (70xx), from 5 email addresses.

## File Structure
- `app/page.tsx` — main dashboard (client component)
- `components/charts/` — TimeSeriesChart, CountryChart, PaymentMethodChart, AmountChart
- `components/filters/FilterPanel.tsx` — sidebar filters
- `components/table/TransactionTable.tsx` — paginated sortable table
- `components/ui/` — Badge, StatsBar, TransactionDetail
- `lib/types.ts` — TypeScript interfaces
- `lib/data.ts` — data loading utilities
- `lib/utils.ts` — filtering, formatting, groupBy
- `scripts/generate-data.js` — test data generator
